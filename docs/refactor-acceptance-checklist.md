# dg_admin_web_next 重构验收清单

## 1. 试点页原则

不要一开始就全量返工。先选一个重构效果最差、但业务形态具有代表性的页面做试点，先验证输入包、提示词和约束方式是否有效。

推荐顺序：

1. 列表页或表格页
2. 带查询区和分页的页面
3. 避免先拿超复杂的可视化页或编辑器页做试点

## 2. 结构验收

检查以下项目：

- 页面是否放在正确目录
- 路由是否进入 `src/router/modules`
- `meta.title`、`meta.locale`、`meta.type` 是否完整
- 接口是否进入 `src/services/modules`
- 页面是否复用了当前布局体系
- 页面临时状态是否仍留在页面内部
- 是否补充了国际化文案
- 是否接入了现有权限模型

以下情况判定为不合格：

- 页面直接写裸请求
- 单页新增全局 store
- 菜单高亮靠页面手写逻辑维护
- 新增页面绕开现有布局壳

## 3. 视觉验收

检查以下项目：

- 页面是否仍是中后台 `el-card` 骨架
- 查询区、工具区、内容区、分页区是否层级清晰
- 是否沿用现有 `var(--el-*)` 变量
- 间距是否保持在现有项目节奏内，常见为 `8 / 16 / 24 / 32`
- 表格、按钮、tag、链接操作是否接近参考页写法
- 是否没有出现不必要的大面积渐变、发光、装饰图层

以下情况判定为不合格：

- 页面明显像品牌官网、落地页或数据大屏
- 配色与当前项目主色和变量体系脱节
- 使用大量内联样式才能维持布局
- 页面样式靠全局覆盖 Element Plus 才能成立

## 4. 响应式验收

至少检查：

- 桌面端是否可读
- 中小屏下查询区是否能换行或纵向排列
- 表格、卡片、弹窗是否横向溢出
- 按钮区是否在窄屏下仍可操作

以下情况判定为不合格：

- 只有桌面端勉强可用
- 依赖固定宽度导致窄屏错位
- 表单和按钮区在移动端挤压重叠

## 5. 代码验收

检查以下项目：

- 是否复用了现有 `Ec*` 组件或 Element Plus 能力
- 是否引入了新的全局视觉常量或主题常量
- 是否把局部样式写成了局部 `scoped lang="scss"`
- 是否尽量减少了 `:deep` 和内联样式
- 是否输出了“本次复用了哪些现有实现”

以下情况判定为不合格：

- 明显重复造轮子
- 样式逻辑分散在多个无关文件
- 新增新的基础 UI 抽象层
- 输出里说不清参考了哪些现有实现

## 6. 试点页对照检查

每个试点页至少做以下三类对照：

1. 与 `src/views/table/tableBasic.vue` 或 `src/views/list/listBasic.vue` 对照，确认页面骨架、按钮区、分页、表单密度是否一致
2. 检查是否只使用现有布局壳和服务层，没有出现新 `axios`、新全局 store、新主题常量
3. 检查窄屏下查询区、按钮区、表格或卡片是否正常换行，无横向溢出

只有试点页通过后，才建议把同一套提示词和验收清单复制到其他页面。
