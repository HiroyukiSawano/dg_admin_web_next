# dg_admin_web_next 页面骨架范式手册

## 1. 目的

本文档定义新增业务页时优先采用的页型范式，适用于资产管理系统等新模块开发。

推荐参考页：

- `src/views/table/tableBasic.vue`
- `src/views/list/listBasic.vue`
- `src/views/form/formBasic.vue`
- `src/views/form/formGrid.vue`
- `src/views/table/tableOperate/index.vue`

## 2. 标准台账列表页

适用场景：

- 资产台账
- 设备清单
- 出入库记录
- 借还记录

推荐结构：

- `el-card`
- 顶部查询区
- 工具按钮区
- `el-table`
- 底部分页

必须同步补：

- route
- list service
- i18n
- 权限按钮

不应出现：

- 把详情表单直接塞进列表页
- 筛选和工具栏混成一整块无层级区域

## 3. 树 + 表联动页

适用场景：

- 资产分类树 + 资产列表
- 组织树 + 资产归属列表

推荐结构：

- 左侧树区域
- 右侧 `el-card` 列表区
- 树选择影响右侧查询参数

必须同步补：

- 树查询 service
- 列表 service
- 默认选中策略
- 空态与无树节点兜底

不应出现：

- 树和表都写在一个巨大组件里不拆逻辑
- 左右两侧各自维护不同的查询源

## 4. 主从详情页

适用场景：

- 资产详情 + 关联记录
- 设备详情 + 维护历史

推荐结构：

- 顶部基础信息卡
- 中部详情分组
- 底部关联记录列表

必须同步补：

- detail service
- 关联记录 service
- i18n

不应出现：

- 所有字段平铺成一个超长表单
- 为了详情页单独造一套布局壳

## 5. Tab 化详情页

适用场景：

- 资产详情中存在明显分域，例如基础信息、生命周期、附件、操作日志

推荐结构：

- 顶部基础摘要
- 下方 `el-tabs`
- 每个 tab 内仍保持卡片或表格区分层

必须同步补：

- tab 数据来源定义
- 首屏默认加载策略
- 切换时是否延迟加载

不应出现：

- tab 内重复一套页面级查询区
- 每个 tab 都独立维护不同详情主对象

## 6. 弹窗表单页

适用场景：

- 新增资产
- 编辑基础属性
- 分配归属
- 调拨、报废、借还等动作

推荐结构：

- `el-dialog` 或 `el-drawer`
- 上部表单区
- 下部主次操作按钮

必须同步补：

- 表单默认值
- 校验规则
- 提交 service
- 成功后的刷新动作

不应出现：

- 在弹窗中承载超长、多 tab、多子列表的复杂流程

## 7. 选型规则

新增页面时，先选最接近的范式，再决定细化实现：

1. 先判断是不是列表为主
2. 再判断是否需要树联动
3. 再判断详情是轻量弹窗还是独立页
4. 最后决定表单是否适合弹窗

对于资产管理系统，默认优先：

- 台账列表页
- 树 + 表联动页
- 主从详情页
- 弹窗表单页

只有信息层级很清晰时，才上 tab 化详情页。
